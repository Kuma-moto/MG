# MG — генерация 2D top-view карт в CARLA

![Python](https://img.shields.io/badge/Python-3.x-blue)
![CARLA](https://img.shields.io/badge/CARLA-0.9.13-orange)
![Status](https://img.shields.io/badge/status-research%20prototype-lightgrey)

**MG** — исследовательский инструмент для симулятора [CARLA](https://carla.org/), который расширяет стандартный `manual_control.py` и добавляет возможность получать 2D-вид сверху на дорожную сцену. Проект предназначен для генерации синтетических изображений дорожной инфраструктуры и маршрута автомобиля, которые можно использовать в задачах компьютерного зрения, локализации, навигации и обучения моделей для беспилотного транспорта.

Идея проекта описана в статье **«Создание 2D карты местности в симуляторе Carla»**: вместо подключения дополнительной внешней платформы проект добавляет top-view представление непосредственно в CARLA и позволяет автоматически сохранять изображения во время движения автомобиля по карте в режиме автопилота.

## Возможности

- генерация 2D-карты дорожной сцены на основе карты CARLA/OpenDRIVE;
- отображение дорог, полос, разметки, тротуаров, парковочных зон, дорожных знаков и светофоров;
- интеграция режима вида сверху в модифицированный клиент CARLA `manual_control.py`;
- запуск автомобиля в режиме автопилота по предустановленным картам CARLA;
- запись последовательностей изображений на диск;
- защита от сохранения неинформативных кадров в начале движения: запись кадров камеры выполняется только после начала движения автомобиля;
- постобработка сохранённых карт: синхронизация пар изображений, поиск положения автомобиля и кадрирование карты в единый формат.

## Структура репозитория

```text
MG/
├── map_grab.py    # основной CARLA-клиент для отображения и записи top-view/камерных данных
└── clean_img.py   # скрипт очистки, синхронизации и кадрирования сохранённых изображений карты
```

После запуска скрипты используют следующую структуру каталогов:

```text
Map Projection/
├── img/        # кадры с камеры CARLA
├── map_img/    # сохранённые изображения 2D-карты
└── map_clean/  # очищенные и обрезанные изображения карты
```

> Пути в коде частично заданы жёстко. Перед запуском проверьте пути к CARLA API и каталогам изображений в `map_grab.py` и `clean_img.py`.

## Требования

- Python 3.x;
- CARLA Simulator 0.9.13;
- PythonAPI CARLA;
- `pygame`;
- `numpy`;
- `pandas`;
- `opencv-python`.

Установить Python-зависимости можно так:

```bash
pip install pygame numpy pandas opencv-python
```

## Подготовка

1. Установите CARLA и убедитесь, что PythonAPI доступен из Python.
2. Проверьте путь к CARLA API в `map_grab.py`:

   ```python
   sys.path.append('C:/CARLA_0.9.13/PythonAPI/carla')
   ```

   При необходимости замените его на путь к вашей установке CARLA.

3. Создайте каталоги для выходных данных:

   ```text
   Map Projection/img/
   Map Projection/map_img/
   Map Projection/map_clean/
   ```

4. Если используется `clean_img.py`, проверьте пути:

   ```python
   PATH_MAP = 'C:/SelfDrive/Map Projection/map_img/'
   PATH_IMG = 'C:/SelfDrive/Map Projection/img/'
   PATH_MAP_CLEAN = 'C:/SelfDrive/Map Projection/map_clean/'
   ```

## Запуск

Сначала запустите сервер CARLA.

Windows:

```bash
CarlaUE4.exe
```

Linux:

```bash
./CarlaUE4.sh
```

Затем запустите клиент:

```bash
python map_grab.py --map Town05 --res 640x480
```

Полезные параметры:

```bash
python map_grab.py --map Town05 --res 640x480 --sync
python map_grab.py --map Town05 --show-triggers --show-spawn-points
python map_grab.py --host 127.0.0.1 --port 2000
```

По умолчанию используется карта `Town05`, разрешение `640x480`, роль автомобиля `hero`, включён режим автопилота и предварительная очистка сцены от старых сенсоров/автомобилей.

## Управление

Основные клавиши в окне CARLA-клиента:

| Клавиша | Действие |
|---|---|
| `W`, `A`, `S`, `D` / стрелки | ручное управление автомобилем |
| `P` | включить/выключить автопилот |
| `N` или <code>`</code> | переключить сенсор |
| `1`–`9` | выбрать сенсор по номеру |
| `M` | включить/выключить запись карты, когда выбран режим карты |
| `R` | включить/выключить запись кадров камеры |
| `C` | сменить погоду |
| `F1` | включить/выключить HUD |
| `H` или `?` | показать справку |
| `Esc` | выход |

Для генерации данных обычно достаточно выбрать режим карты/top-view, включить автопилот и активировать запись. Кадры камеры сохраняются только после начала движения автомобиля, что помогает избежать обрезанных кадров на нулевой скорости.

## Постобработка изображений

После записи данных запустите:

```bash
python clean_img.py
```

Скрипт выполняет три операции:

1. сравнивает файлы в `img/` и `map_img/`;
2. удаляет несинхронизированные изображения, у которых нет пары;
3. находит зелёный маркер автомобиля на карте, кадрирует область вокруг него и сохраняет результат в `map_clean/`.

Параметры кадрирования задаются в начале `clean_img.py`:

```python
PIXELS_FROM_MAP_BOTTOM = 210
PIXELS_FROM_CENTRE_SIDEWISE = 10
CROP_MAP_PIXELS_Y = 264
CROP_MAP_PIXELS_X = 240
```

## Как это работает

`map_grab.py` строит 2D-представление карты CARLA на основе OpenDRIVE-данных: получает waypoint-сеть, рассчитывает границы карты, переводит мировые координаты в пиксели и отрисовывает дорожную инфраструктуру средствами `pygame`. Для ускорения повторных запусков карта кэшируется в каталоге `cache/no_rendering_mode` по хэшу OpenDRIVE-содержимого.

Далее скрипт создаёт автомобиль, подключает стандартные датчики CARLA, позволяет переключаться между сенсорами и отображает карту поверх основного окна. Для маршрутизации используется `GlobalRoutePlanner`, который может выбирать случайный достаточно длинный маршрут между точками спавна.

`clean_img.py` нужен как отдельный этап подготовки датасета: он выравнивает сохранённые карты по положению автомобиля, чтобы изображения имели одинаковую область интереса и были удобны для дальнейшего обучения моделей.

## Возможные сценарии применения

- подготовка синтетических данных для обучения нейросетей;

- эксперименты с представлением сцены с высоты птичьего полёта без подключения сторонних платформ.

## Известные ограничения

- проект рассчитан на локальную установку CARLA и требует корректной настройки путей;
- часть путей в коде задана под Windows;
- `clean_img.py` ожидает конкретную структуру папок и наличие зелёного маркера автомобиля на карте;
- репозиторий является исследовательским прототипом и может потребовать адаптации под другую версию CARLA или структуру проекта.
